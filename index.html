mysql 基础

小知识点

  • mysql目前属于Oracle公司，它是一个开源的关系型数据库管理系统，分为社区版和企业版
  • mysql的目录结构：
    • bin目录，存储可执行文件
    • data目录，存储数据文件
    • docs目录，文档
    • include目录，存储包含的头文件’
    • lib目录，存储库文件
    • share目录，错误信息和字符集文件
  • set names gbk;
  • DELIMITER // 用//结束每句话
  • 图形化管理工具Navicat for MySQL，PHPMyAdmin,MySQL Workbench

    存储引擎

  • MySql可以将数据以不同的技术存储在文件（内存）中，这种技术成为存储引擎
  • 存储引擎，可以通过配置文件修改，也可以在创建表时最后加ENGINE=MyISAM;,还可以alter修改：
    • MyISAM:256TB，支持索引，表锁，支持数据压缩
    • InnoDB:64TB，支持事务处理，支持索引，行锁，支持外键
    • Memory:有限制，支持索引，表锁
    • Archive:无限制，行锁，支持数据压缩
    • CSV:不支持索引

      安装配置

  • MSI安装/ZIP安装，官网下载即可
  • 安装类型分三种：Typical安装（典型安装）、Custom（自定义安装）、Complete（完全安装），通常用典型安装就可以
  • bin目录下有配置文件的exe文件，按提示操作即可（标准配置）
  • 配置目录下，min.ini文件–修改编码方式[mysql] default-character-set=utf8 [mysqld] character-set-server=utf8
  • 命令行中启动/停止Mysql服务（net start/stop mysql）
  • 命令行中登录 mysql -uroot -p -P3306 -h127.0.0.1 –prompt \h
    • u用户名，p密码，P端口号，D打开指定数据库，h服务器名称,V输出版本信息并且退出，–prompt \h(服务器名称)\D(完整的日期)\d(当前数据库)\u(当前用户)提示符
  • 退出 exit;/quit;/\q;
  • 显示当前服务器版本/当前日期时间/当前用户（SELECT VERSION()/NOW()/USER();）

    使用

  • Mysql语句的规范（约定俗成，大小写混也不是不行）
    • 关键字和函数名称全部大写
    • 数据库名称、表名称、字段名称全部小写
    • SQL语句必须分号结尾

      数据库操作

  • 创建数据库 CREATE DATABASE [IF NOT EXISTS] t1 [CHARACTER SET gbk];
  • 查看数据库 SHOW [CREATE] DATABASE [db_name];
  • 修改数据库 ALTER DATABASE db_name CHARACTER SET[=]gbk
  • 删除数据库 DROP DATABASE db_name;

数据表常规操作

  • 数据类型：
    • 整型：TINTINT(1字节)，SMALLINT(2),MEDIUMINT(3),INT(4),BIGINT(8)
    • 浮点型：FLOAT[(M,D)],DOUBLE[(M,D)] M:数字总位数 D:小数点后面的位数
    • 日期时间类型：YEAR,TIME,DATE,DATETIME,TIMESTAMP（时间戳）
    • 字符型 CHAR(M),VARCHAR(M),TINYTEXT,TEXT,MEDIUMTEXT,LONGTEXT,枚举（选择）：ENUM(‘value1’,’’,…),SET(‘value1’,’’,…)
  • 表操作
    • USE 数据库名称;
    • CREATE TABLE [IF NOT EXISTS] NAME(age INT NOT NULL,id int AUTO_INCREMENT(自动编号1++，必须定义成主键(会自动产生索引)) PRIMARY KEY(主键),…)
    • SHOW TABLES[FROM db_name][link’pattern’|WHERE expr] 查看数据表结构(SHOW COLUMNS FROM db_name)
    • 唯一约束 UNIQUE KEY,可以存在空值
    • 默认约束 DEFAULE （sex ENUM(‘1’,’2’) DEFAULE’1’;）
    • 外键约束 FOREIGN KEY （pid）REFERENCES table1 (id) [ON DELETE CASCADE]
      • CASCADE:从父表删除或更新且自动删除或更新子表中匹配的行
      • SET NULL:从父表删除且更新行，并设置子表中的外键列为null，如果使用该选项，必须保证子表列没有指定NOT NULL
      • RESTRICT:拒绝对父表的删除更新操作
      • 保持数据一致性、完整性，实现数据表之间的链接，这也是称为关系型数据库的原因
      • 父表和子表必须使用相同的存储引擎（InnoDB）可在配置文件中修改，而且禁止使用临时表
      • 外键列和参照列必须具有相似的数据类型
      • 外键列和参照列必须创建索引，如果外键列不存在索引的话，mysql将自动创建索引
    • 修改数据表
      • 添加单列 ALTER TABLE t1 ADD age int [约束] [FIRST|AFTER 某列名]
      • 添加多列 ALTER TABLE t1 ADD（age int,,,）
      • 删除列 ALTER TABLE t1 DROP col_name,DROP col_name
      • 添加约束 ALTER TABLE t1 ADD[CONSTRAINT PK_username自定义约束名]PRIMARY KEY(id)
      • 删除索引 ALTER TABLE t1 DROP index(pid)
      • 修改列定义 ALTER TABLE t1 MODIFY age smallint [位置]
      • 修改列名称 ALTER TABLE t1 CHANGE old_name new_name column_definition [位置]
      • 修改表名 ALTER TABLE t1 RENAME t2 (最好别改，牵一发动全身)
  • 记录操作
    • INSERT tbl_name[(col_name,…)]VALUES(val,…);
    • INSERT tbl_name SET col_name=’’,id=’’,… //可以使用子查询
    • INSERT tbl_name[(col_name,…)]SELECT… //把查找结果插入
    • UPDATA users set age=age+5,sex=’2’[WHERE…]; //更新记录
    • DELETE FROM tbl_name [WHERE id=6 AND…]; //删除记录
    • SELECT *[可以是某些列，逗号隔开][AS 别名] FROM tbl_name [WHERE…]；
      • group by{col_name|position}[ASC|DESC]…:查询结果分组
      • having age>35:分组条件,卸载group by 列的后面，且age必须在查询结果中
      • order by{col_name|expr|position}[ASC|DESC]…:对查询结果进行排序
      • limit [起始位置0开始],2(记录数):限制查询结果返回的数量

  • 字符函数

  • 比较运算符与函数

    -[NOT]IN():是否在列出值范围内In(5,15,20,30)

    MYSQL存储过程

  • 存储过程是SQL语句和控制语句的预编译集合，以一个名称存储并作为一个单元处理
  • 优点：增强sql语句的功能和灵活性，实行较快的执行速度，减少网络流量
CREATE PROCEDURE sp1() select version();
CALL sp1[()]

DELIMITER //
CREATE PROCEDURE removeUserById(IN id1 INT UNSIGNED)
BEGIN
DELETE FROM users where id=id1;
END 
//
DELIMITER ;
CALL removeUserById(3);
DROP PROCEDURE removeUserById;

DELIMITER //
CREATE PROCEDURE removeUserByIdAndreturn(IN id1 INT UNSIGNED,OUT usernums int unsigned)
BEGIN
DELETE FROM users where id=id1;
SELECT COUNT(id)FROM users INTO usernums;
END 
//
DELIMITER ;
CALL removeUserByIdAndreturn(3,@nums);
SELECT @nums;

SET @i=8;//用户变量
  • 存储过程参数类型
    • IN：表示该参数的值必须在调用存储过程时指定，不可返回
    • OUT：表示该参数的值可以被存储过程改变，并且可以返回
    • INOUT：表示该参数的值在调用时指定，可以被改变和返回
  • 与自定义函数的区别
    • 存储过程实现功能要复杂一些，函数的针对性更强
    • 存储过程可以返回多个值，函数只能有一个返回值
    • 存储过程一般独立的来执行，函数可以作为其他sql语句的组成部分来出现

c++基础

基础知识

与c语言的小区别

  • 1.c语言没有提供bool类型，c++有bool类型（true,false=0），以此类型输出需加boolalpha
  • 2.两种初始化方法 int x=2 和int x(2),c++可以随用随定义
  • 3.c语言用scanf输入，print输出；c++用cin输入，cout输出。（cout<<x<<endl;）,endl(类似于\n回车)；（cin>>x>>y）
  • 4.八进制输出（cout<<oct<<x）,十进制（dec）,十六进制（hex）
  • 5.面向对象三大特征：封装、多态、继承

其他细节

  • 1.namespace(命名空间)，用来区分来自不同地方的相同名字的变量和函数等
  • 2.#include 头文件 内包含输入输出语法，但是它在空间std中，需要using namespace std;

  • 3.引用类型（变量的别名）
    • 引用必须初始化（int a =2;int &b=a;），对别名的修改会影响到本身
    • 指针类型的引用（*&指针引用名=指针;）int a=10; int *p=&a; int *&q=p; *q=20; cout <<a;—-结果为20
    • 函数参数的引用，比指针更方便（void get(int &x,int &y){}）
  • 4.const
    • const int x=3//常量，无法更改
    • int const p(const int *p)=&x,此时p可以指向另一个地址，const修饰的是p,即x的值
    • int *const p=&x,此时p的指向不能变，x的值可变
    • const int &y=x,此时通过变量名x修改可以，引用名Y不行
    • #define 是宏定义，编译时不检查语法错误，const检查
  • 5.c++中函数参数可以有默认值，但含有默认值的参数要放到后面，使用时最好函数声明时写上默认值，定义时不写，实参会覆盖默认值
  • 6.函数重载：在同一作用域中，函数名相同，函数参数不同
  • 7.内联函数inline：编译时将函数体代码和实参代替函数调用语句(声明：inline int max();)。递归函数无法使用内联函数，内联编译是建议性的，由编译器决定。逻辑简单、调用频繁的函数建议使用内联函数
  • 8.内存管理：内存的申请与释放（int *p=new int;int *arr=new int[10]//申请块内存）（delete p/[]p;p=null）（c中void *malloc()–void free()）
  • 9.String 类型 头文件(#include)
    • 初始化 string name=”dd”;string hobby(“football”);string s2(s1);string s4(n,’c’)//n个c;
    • s.empty()//判断S是否为空串
    • s.size()//返回s中字符的个数
    • 字符串连接中””+s1+””,s1+s2合法，但是””+””不合法
  • 10.内存分区：栈区（变量，指针，由系统控制），堆区（new），全局区（存储全局变量及静态变量），常量区（string），代码区（存储逻辑代码的二进制）

封装（类与对象）

  • 1.类中访问限定符：public（公共），protected（受保护的），private（私有的）
  • 2.实例化对象：

    • 从堆中实例化

  • 3.通常将方法设为公有属性，将数据成员设为私有属性，以便封装数据成员（定义set,get方法除了可以保护数据，还可以对传入的数据做判断）
  • 4.类内定义，在类里定义的函数，编译器会优先将它们作为内联函数（inline）
  • 5.类外定义（函数体写在类外）
    • 同文件类外定义（函数与类（car）在同一文件，void car::run(){}）
    • 分文件类外定义（推荐使用，car.h文件中写类，car.cpp文件中#include”car.h”和方法void car::run(){}）
  • 6.对象初始化，有的数据只需要初始化一次，此时使用构造函数，对象会自动调用
  • 7.构造函数没有返回值，可以重载，且仅会被调用一次。当用户没有定义时，编译器会自动生成一个构造函数，构造函数名称与类名一致
  • 8.初始化列表（student():m_strname(“jim”),m_intAge(10){}）,只能用于构造函数，先于构造函数执行，可以给类中定义的常量（const）初始化赋值
  • 9.拷贝构造函数（student(const student&stu){}）系统会自动生成（拷贝和普通系统只会生成一个）,当采用直接初始化或复制初始化实例对象时，系统自动调用拷贝构造函数
  • 10.析构函数（~类名(){}），对象销毁前自动调用，释放资源。没有自定义的话系统会自动生成，且不允许有参数
  • 11.浅拷贝，当出现指针时，会产生麻烦
  • 12.深拷贝，赋值时新建另一个堆，将每一个值都复制过去
  • 13.this指针，取当前对象的地址（*this是对象）
  • 14.常对象成员（初始化列表赋值）和常成员函数（不可修改成员的值，重载时只有常对象才能调用常成员函数，也只能调用常成员函数。普通对象都可调用）

    继承

  • 1.公有继承（class A:public B{}）,B可以调用A中的属性和方法,继承时该是什么类型就是什么类型
  • 2.保护继承（class A:protected B{}），A中protected和public的成员被继承到B中后是protected类型
  • 3.私有继承（class A:private B{}），A中protected和public成员被继承到B中后是private类型
  • 4.无论哪种继承方式，A中私有属性的成员被继承到B中后处于不可见位置，B无法访问
  • 5.隐藏 子类与父类有同名函数时，从父类继承过来的函数（或数据成员）将被隐藏，但是可以通过特殊方式使用（对象.父类::函数（））
  • 6.虚析构函数（virtual~），父类使用虚析构函数，此时将子类赋值给父类定义的指针的情况下释放指针时不会遗漏子类的析构函数，否则会造成内存泄漏等问题
  • 7.多重继承（子孙关系），一个子类同时有多个父类称为多继承
  • 8.虚继承（：virtual public A()），多继承+多重继承时（菱形继承），子类继承父类时使用虚继承，可以使孙类减少冗余和出现重定义的问题

    多态（虚函数）

  • 1.多态：相同对象收到不同消息或不同对象收到相同消息时产生不同的动作
  • 2.静态多态（早绑定），通常为函数重载
  • 3.动态多态，以封装和继承为基础，父类指针指向子类对象，通过父类指针完成动作（父类的成员函数为虚函数才行）
  • 4.普通函数、静态成员函数（static）、内联函数、构造函数不能是虚函数
  • 5.函数覆盖 虚函数继承后，子类的虚函数表中的内容会覆盖父类中的同名内容，实现多态
  • 6.纯虚函数（virtual double A()=0;）,含有纯虚函数的类为抽象类，抽象类无法实例化对象
  • 7.接口类：仅含有纯虚函数的类
  • 8.RTTI（运行时类型识别）（typeid(obj).name();typeid(*obj)==typeid(类名);Bird *bird=dynamic_cast<Bird>(obj);注：只能用于指针或引用的转换）
  • 9.举例 try{f1()}…catch（int/…）{} f1(){throw 1 }抛出异常
  • 10.常见的异常：数组下标越界、除数为0、内存不足

  • 2.友元关系不能传递，友元关系的单向性

  • 4.运算符重载（类中 对-重载：类名& operator-();-对象名();）,-(取反),++,–
  • 5.二元运算符重载，A& operator+(A &a1,A &a2);friend ostream& operator<<(ostream &out,const A &a1);int operator[](int index);

urllib是标准库，它是一个工具包模块

HTTPS证书忽略

  • HTTPS使用SSL安全套阶层协议，在传输层对网络数据进行加密。HTTPS使用时需要证书，而证书需要CA认证
  • CA（certificate authority）是数字证书认证中心的简称，是受信任的第三方。是指发放、管理、废除数字证书的的机构
  • 当爬取没有认证的网址时，会报错，例如12306，它有自己给自己的根证书，正常情况下需要下载证书才能访问。
  • import ssl 包中有方法可以模拟浏览器忽略证书不安全信息 
context = ssl._create_unverified_context()#忽略不信任证书
res = urlopen(request,context=context)

Markdown语法

Markdown是一种纯文本格式的标记语言。通过简单的标记语法，它可以使普通文本内容具有一定的格式。

相比WYSIWYG编辑器

优点：

  • 因为是纯文本，所以只要支持Markdown的地方都能获得一样的编辑效果，可以让作者摆脱排版的困扰，专心写作。
  • 操作简单。比如:WYSIWYG编辑时标记个标题，先选中内容，再点击导航栏的标题按钮，选择几级标题。要三个步骤。而Markdown只需要在标题内容前加#即可

缺点：

  • 需要记一些语法（当然，是很简单。五分钟学会）。
  • 有些平台不支持Markdown编辑模式。

一、标题

  • 在想要设置为标题的文字前面加#来表示

  • 一个#是一级标题，二个#是二级标题，以此类推。支持六级标题。

  • 注：标准语法一般在#后跟个空格再写文字，示例不加空格，效果图加空格
    示例：
    #这是一级标题
    ##这是二级标题
    ###这是三级标题
    ####这是四级标题
    #####这是五级标题
    ######这是六级标题
    效果如下：

    这是一级标题

    这是二级标题

    这是三级标题

    这是四级标题

    这是五级标题
    这是六级标题

二、字体

  • 加粗：要加粗的文字左右分别用两个*号包起来

  • 斜体：要倾斜的文字左右分别用一个*号包起来

  • 斜体加粗：要倾斜和加粗的文字左右分别用三个*号包起来

  • 删除线：要加删除线的文字左右分别用两个~~号包起来

效果：
这是加粗的文字
这是倾斜的文字`
这是斜体加粗的文字
这是加删除线的文字

五、图片

  • 语法：![图片alt](图片地址 ‘’图片title’’)

  • alt就是显示在图片下面的文字，相当于对图片内容的解释。

  • title是图片的标题，当鼠标移到图片上时显示的内容。title可加可不加
    示例：

八、表格

  • 语法：

    表头 表头 表头
    内容 内容 内容
    内容 内容 内容

  • 第二行分割表头和内容。

  • 有一个就行，为了对齐，多加了几个

  • 文字默认居左

  • 两边加：表示文字居中

  • 右边加：表示文字居右

  • 注：原生的语法两边都要用 | 包起来。此处省略
    示例：
    姓名|技能|排行

  • -|:–:|–:
    刘备|哭|大哥
    关羽|打|二哥
    张飞|骂|三弟

九、代码

  • 语法：单行代码：代码之间分别用一个反引号包起来,多行’’’包起来，最方便的是将代码选中再按tab键，前后空三行左右
